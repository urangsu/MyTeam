# MyTeam 용어 정책 (Terminology Policy)

> Round 137A-145Z 제정. 앱 내 모든 레이블·안내 문구·코드 주석은 이 정책을 따른다.

## 공식 용어 대조표

| 구 표현 (금지) | 공식 표현 | 적용 위치 |
|---|---|---|
| 채팅방 | 워크룸 | TeamStatusView, AgentChatView, 안내 문구 전체 |
| 프로젝트 | 대화 (사이드바 헤더) | AgentChatView 사이드바 |
| 기본 프로젝트 | 워크룸 1 | 기본 방 생성 시 이름 |
| 스케줄 근무 | 예약 작업 | TeamStatusView, schedulePopupCard, 안내 문구 |
| 스케줄 | 예약 작업 | 버튼 레이블, 헬프 텍스트 |
| 프로젝트 N | 워크룸 N | 새 방 생성 시 기본 이름 |

## 이유

- "채팅방"은 카톡/슬랙 연상 → 단순 메시지 교환 도구처럼 보임
- "워크룸"은 업무 공간 개념 → 팀이 함께 작업하는 맥락에 적합
- "스케줄 근무"는 교대근무 연상 → "예약 작업"이 자동화 의미 전달
- "프로젝트"는 대규모 프로젝트 관리 연상 → "대화"가 에이전트별 1:1 맥락에 적합

## 코드 적용 위치 (Round 137A 완료)

- `TeamStatusView.swift`: 채팅방→워크룸, 스케줄 근무→예약 작업
- `AgentChatView.swift`: 프로젝트→대화 (사이드바 헤더)
- `AgentWindowManager.swift`: "기본 프로젝트"→"워크룸 1"
- `TeamStatusView.swift`: createRoom 기본 이름 "워크룸 N"

## 신규 UI 개발 시 체크리스트

- [ ] "채팅방" 문자열 없음
- [ ] "스케줄 근무" 문자열 없음
- [ ] 새 방 기본 이름: "워크룸 N"
- [ ] 예약 작업 entry point: 단일 경로
