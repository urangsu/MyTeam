## Content Rules

### Must Include
- ✅ 실제 앱 화면 캡처 (2-3개)
- ✅ 첫 100자 안에 핵심 키워드 포함
- ✅ "회의록", "체크리스트", "macOS AI" 자연스럽게 반복
- ✅ H1/H2 구조로 SEO 최적화
- ✅ Meta description 60자 내외
- ✅ Internal links (다른 MyTeam 포스트 연결)

### Avoid
- ❌ "외부 서버 없음" 과장 표현
- ❌ "완전 로컬" 단정
- ❌ Gmail/Naver/Calendar write 미구현 기능 홍보
- ❌ 과장된 시간 절감 주장 (숫자로 입증 없이)
- ❌ 다른 도구 비하

### Tone & Style
- 친근하고 직설적 (우 존댓말)
- 1인칭 ("저는...", "우리는...")
- 실제 사용 사례 중심
- 번역투 피하기

## SEO Metadata Template

### Blog Post Template
```
Title: [메인키워드 + 보조키워드 + 설명]
Meta Description: [60자, 첫 문장 요약, 메인키워드 포함]
Keywords: [메인 3-5개 + 보조 3-5개]
Category: MyTeam, macOS, AI도구, 생산성
Internal Links: [다른 MyTeam 포스트 2-3개]
Related Posts: [최대 3개]
```

```
Title: 맥북에서 회의록 자동으로 만드는 법 — MyTeam 사용기 (2025)

```

## Naver Blog + Blind Strategy

### Naver Blog Features
- 구독자 확보 (구독 CTA 매 포스트)
- 시리즈 연결 (12개 포스트 3-4개 시리즈로 묶기)
- 카테고리 정리 (MyTeam, macOS, AI도구, 생산성 4개)
- 댓글 응답 (첫 48시간 빠른 응답)

### Blind Strategy
- "1인 창업자 커뮤니티"에서 자연스러운 언급
- 스크린샷 공유 (미리 Blind 커뮤니티에 공개 피드백)
- "이걸 써봤는데..."로 시작하는 자연스러운 후기
- 과도한 자극적 카피 금지

### Product Hunt Strategy
- 한국 시간대가 아닌 UTC 기준 화요일/수요일 런칭
- "내 컴퓨터 안의 AI 팀" 포지셔닝 명확히
- 스크린샷: 회의록 artifact + next actions 우선
- 한국 커뮤니티 관계자에게만 사전 공지
