## 시나리오 1: 초보자 예시 플로우 (API 키 없음)

### 목표
API 키 없이도 치코 예시 업무 문서가 생성되고, 다음 액션이 표시되는지 확인한다.

### 클릭 순서

1. **앱 실행** → 첫 화면 확인
   - [ ] `OnboardingCardView` 또는 `FirstLaunchBannerView`가 1개만 표시되는가?
   - [ ] `LocalOnlyModeCardView`와 `FirstLaunchBannerView`가 동시에 표시되지 않는가?

2. **워크룸 메인 화면** → 초보자 카드 확인
   - [ ] "예시로 먼저 해보기" 카드가 표시되는가?
   - [ ] 카드에 치코 아이콘이 포함되어 있는가?

3. **"예시로 먼저 해보기" 탭**
   - [ ] 로딩 인디케이터가 표시되는가? (짧게)
   - [ ] 치코 메시지: "회의록 예시를 만들어봤어요" 또는 유사 안내가 나타나는가?
   - [ ] Artifact 카드가 채팅 스트림 하단에 나타나는가?

4. **Artifact 카드 확인**
   - [ ] 파일명이 `예시_회의록_*.md` 형식인가?
   - [ ] "Finder에서 열기" 버튼이 동작하는가?
   - [ ] Finder에서 실제 파일이 열리는가?
   - [ ] 파일 내용이 한국어 마크다운 회의록인가?

5. **다음 액션 표시**
   - [ ] Artifact 생성 후 `WorkroomNextAction` 버튼들이 나타나는가?
   - [ ] "다음 문서 만들기", "이 문서 수정하기" 등 후속 액션이 1개 이상 표시되는가?

### 합격 기준
- API 키 없이 Artifact 1개 생성 완료
- 다음 액션 버튼 1개 이상 표시
- 오류 메시지 미표시

### 상태: `[ ] pending`

### 클릭 순서

1. **워크룸 열기** → 치코 캐릭터 표시 확인

3. **메시지 전송** (로컬 스킬 or 예시 실행)
   - [ ] 치코가 `thinking` 상태로 전환되는가?

4. **워크플로우 완료**
   - [ ] 치코가 `greeting` 또는 `joy` 상태로 전환되는가?

5. **앱 최소화 → 복귀**

## 시나리오 3: Friendly Recovery — 파일 손상/삭제

### 목표
Artifact 파일이 삭제되거나 손상됐을 때 기술 용어 없이 복구 안내가 표시되는지 확인한다.

### 사전 준비
- 시나리오 1 완료 → Artifact 1개 이상 존재
- 생성된 `예시_회의록_*.md` 파일의 실제 경로를 Finder에서 확인해 둠

### 클릭 순서

1. **Finder에서 파일 삭제**
   - Artifact 파일을 Trash로 이동 (완전 삭제 아님)

2. **앱에서 Artifact 카드 확인**
   - [ ] 카드 상태가 변경되는가?
   - [ ] "파일을 열 수 없음" 또는 "파일이 삭제되었습니다" 메시지가 표시되는가?
   - [ ] "경로 오류", "해시 불일치" 같은 기술 용어가 **표시되지 않는가**?

3. **복구 버튼 확인**
   - [ ] "다시 만들기" 또는 "위치 변경" 버튼이 표시되는가?
   - [ ] 버튼이 오렌지 계열 색상인가? (경고 톤)

4. **"다시 만들기" 탭**
   - [ ] 앱 크래시 없음
   - [ ] 외부 쓰기(메일 발송, 캘린더 등록) 없음
   - [ ] 토스트 또는 상태 메시지 표시

5. **파일 해시 불일치 시뮬레이션** (선택)
   - 파일 내용을 텍스트 에디터로 수정 후 저장
   - [ ] "파일이 변경됐습니다" 메시지 표시 여부 확인
   - [ ] 기술 용어 없음 확인

### 합격 기준
- 기술 용어 ("경로 오류", "해시 불일치") 미표시
- 복구 버튼 동작 중 외부 쓰기 없음
- 앱 크래시 없음

### 상태: `[ ] pending`
